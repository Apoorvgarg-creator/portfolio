# Lessons from Open Source

Open source has taught me more about software engineering than any job or course. Here are the biggest lessons.

## 1. Read Before You Write

The best contributors spend 80% of their time reading existing code before writing a single line. Understanding the codebase's patterns, conventions, and constraints will make your contribution fit naturally.

## 2. Small PRs Win

A 50-line PR gets reviewed in an hour. A 500-line PR sits for weeks. Break your work into small, reviewable chunks.

## 3. Documentation Is a Feature

The best code in the world is useless if nobody knows how to use it. Write docs like someone's first day depends on them — because it does.

## 4. Maintainer Burnout Is Real

If you maintain a project, set boundaries early. It's okay to say no. It's okay to take breaks. The project will survive.
